使用指南

三步审查工作流

V5.3：先审查、后脱敏、可还原。预审与实际脱敏使用统一 ONNX NER 引擎，支持自定义实体即时反馈和天干假名命名。

01

上传文件

将 .txt / .docx / .pdf 文件拖入窗口或点击选择。引擎自动识别格式并提取全文内容，支持法律合同、起诉状、证据材料等。

02

预扫描审查 + 脱敏

统一 ONNX BERT-NER 引擎扫描全文，列出检测到的敏感实体。添加自定义实体（人名/公司名/通用），所有操作即时反馈。一键执行脱敏。

03

保存 + 按需还原

保存脱敏文件和 JSON 映射表。需要还原时，切换到还原 Tab，上传脱敏文件 + 映射表，一键还原为原始文本。

脱敏思路

五层递进，AI 辅助，可逆闭环

life法 采用五层递进式策略，从精确字符串匹配到 AI 上下文理解再到还原闭环，逐层深入，兼顾精度、召回与安全性。

1

精确字符串替换

自定义优先

用户手动指定的实体在所有规则引擎启动前先执行精确全字匹配替换。人名 → 甲某/乙某/丙某，公司名 → 公司A/公司B/公司C。V5.3 新增通用自定义：自由定义查找/替换文本。

精确全字匹配 代号预分配 跨段落一致 通用自定义 持久化保存
2

正则规则引擎

可配置 · 高精度

8 种简单规则通过 JSON 配置文件驱动，无需改代码即可增删调整。支持「前N位保留 + 中间打码 + 后M位保留」的灵活脱敏格式。每条规则独立启用/禁用。

身份证号 · 前6+****+后4 手机号 · 前3+****+后4 统一社会信用代码 邮箱 · 座机号 日期 · 金额 JSON 热配置
3

上下文感知规则 + ONNX BERT-NER

核心引擎 · 分块推理

人名识别采用ONNX BERT-NER 深度学习模型，100-token 滑动窗口分块推理（20 重叠），解决 BERT 长文本注意力退化。公司名、机构名、地址和银行账号均有专用上下文规则。

人名 · BERT-NER 分块推理 100-token 窗口 + 20 重叠 332 假阳性黑名单 公司名 · 全称+简称关联 机构名 · 银行/政府分类 地址 · 省市保留+细节打码
4

精确格式回写 + PDF 遮罩

docx · PDF · run 级别精度

对于 .docx 文件，使用 difflib.SequenceMatcher 精确追踪变更区间，间距 ≤4 字符的碎片自动合并，逐 run 回写到 Word XML 结构。PDF 生成黑色遮罩覆盖敏感信息位置。

difflib 精确区间追踪 碎片合并（≤4字符） run 级别 XML 回写 PDF 黑色遮罩覆盖 表格 + 段落全覆盖 跨段落代号一致
5

回溯还原闭环 V5.2 新增

脱敏可逆 · JSON 驱动

脱敏不是单向的——life法生成完整 JSON 映射表记录每一条替换关系。需要还原时，上传脱敏文件 + 映射表即可一键还原全部敏感信息。支持 .txt 和 .docx 格式，还原后保留原始排版。

JSON 映射表驱动 一键全量还原 txt + docx 双格式 Tab 切换 UI 还原后格式保留 安全闭环链路
