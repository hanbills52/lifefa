使用指南

三步审查工作流

V4.0 全新设计：先审查、后脱敏。在脱敏前就能看到所有将被处理的敏感信息。

01

上传文件

将 .txt 或 .docx 文件拖入窗口，或点击按钮选择文件。引擎自动识别文件格式并提取全文内容。

02

预扫描审查

引擎自动扫描全文，列出所有检测到的敏感实体。您可以添加漏检项、标记排除项、增删自定义规则，确认无误后再进入脱敏。

03

确认脱敏

一键执行脱敏，双栏并排对比原文和脱敏结果。确认无误后保存脱敏文件和可回溯映射表，.docx 格式完整保留。

脱敏思路

分层递进，AI 辅助

life法 采用四层递进式脱敏策略，从精确字符串匹配到 AI 上下文理解，逐层深入，兼顾精度与召回。

1

精确字符串替换

自定义优先

用户手动指定的实体在所有规则引擎启动前先执行精确全字匹配替换。人名统一分配为「人员A / 人员B / 人员C」，公司名统一分配为「公司A / 公司B / 公司C」，保证全文代号一致。

精确全字匹配 代号预分配 跨段落一致 持久化保存
2

正则规则引擎

可配置 · 高精度

7 种简单规则通过 JSON 配置文件驱动，无需改代码即可增删调整。支持「前N位保留 + 中间打码 + 后M位保留」的灵活脱敏格式。每条规则独立启用/禁用，用户可按需自由组合。

身份证号 · 前6+****+后4 手机号 · 前3+****+后4 统一社会信用代码 邮箱 · 座机号 日期 · 金额 JSON 热配置
3

上下文感知规则 + AI 混合识别

核心引擎 · BERT-NER + 规则双通道

5 条复杂规则由 Python 函数实现，具备上下文理解能力。其中人名识别采用双通道混合架构：ONNX BERT-NER 深度学习模型（精准、零误报）与百家姓滑窗 + 职位锚定规则引擎（覆盖、高召回）并行扫描，经三阶层策略合并去重。公司名、机构名、地址和银行账号也均有专用上下文规则。

人名 · ONNX + 规则双通道 324 假阳性黑名单 公司名 · 全称+简称关联 机构名 · 银行/政府分类 地址 · 省市保留+细节打码 银行账号 · 多格式兼容
4

精确格式回写

docx · run 级别精度

对于 .docx 文件，引擎使用 difflib.SequenceMatcher 精确追踪原文与脱敏文本之间的每一个变更区间。间隔 ≤4 字符的碎片变更自动合并为单一区间，然后逐 run 回写到 Word 文档的 XML 结构中，完整保留字体、颜色、加粗、斜体、字号等全部格式。同时支持全文预扫 + 逐段落独立脱敏，保证跨段落公司编号和人名代号的一致性。

difflib 精确区间追踪 碎片合并（≤4字符） run 级别 XML 回写 全文预扫 + 逐段脱敏 跨段落代号一致 表格 + 段落全覆盖
